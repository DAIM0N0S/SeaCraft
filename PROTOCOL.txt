    Сервер
    
Сервер обеспечивает поиск игроков, соединение двух игроков, обмен ударами между игроками, проверку корректности ходов.


    Клиент
    
Клиент обеспечивает пользовательский интерфейс, возможность расстановки кораблей, подключения к серверу и осуществлению матча.


    Сеанс
    
> mbclient:1:SuperBot:119xoOlh%2:
< mbserver:1:
> field:001001011100...001101011:
< found:
< go:
> step:4:3:
< field2:miss:4:3:
< field1:miss:6:8:
< go:
> step:2:7:
< field2:half:2:7:
< go:
> step:2:8:
< field2:miss:2:8:
< field1:half:3:2:
< field1:half:3:3:
< field1:kill:3:4:
< field1:miss:6:4:
< go:
…
< win:
disconnect


    Описание
    
1) Клиент посылает строку: mbclient:версия_протокола_клиента:логин:пароль (протокол - целое положительное число, логин — строка из латинских букв и цифр, пароль — строка из латинских букв, цифр, спецсимволов)
2) Сервер посылает строку: mbserver:версия_протокола_сервера (целое положительное число)
3) Клиент посылает строку: field:расставленные_корабли (1 — есть корабль, 0 — нет корабля, в порядке слева направо, сверху вниз)
4) Сервер посылает команду found как только противник найден.
5) Сервер посылает команду go, которая говорит, что клиенту необходимо сделать ход.
6) Клиент посылает команду step:клетка_по_X:клетка_по_Y (нумерация клеток с нуля до 9 по обоим осям)
7) Сервер посылает команду field2:информация_о_попадании:клетка_по_X:клетка_по_Y. Информация о попадании — один из трёх вариантов: miss – мимо, half – ранил, kill – убил. Информация о поле противника (попал ли игрок по противнику).
8) Сервер посылает команду  field1:информация_о_попадании:клетка_по_X:клетка_по_Y. Информация о поле игрока (попал ли противник по игроку).
9) По окончанию игры сервер посылает команду win или lose и разрывает соединение.
10) В случае если противник покинул игру, сервер посылает команду diconnected: и разрывает соединение.
11) В случае неверного запроса сервер посылает команду wrongcmd:
